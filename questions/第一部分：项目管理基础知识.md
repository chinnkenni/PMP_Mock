# 答案与解析

# 1、答案：A

解析：项目经理不仅应该完成项目任务，还应该满足项目团队和团队成员的需求。 

# 2、答案：B

解析：过程组不应与项目阶段混淆。每个阶段都应该包含所有的过程组。 

# 3、答案：D

解析：基于项目计划和工作绩效数据，分析整理出工作绩效信息。 

# 4、答案：A

解析：会计业务通常是不断重复进行的，是运营 

# 5、答案：A

解析：项目经理的角色可能因组织而异。在一些组织中，项目经理可能会在项目启动前参与评估和分析活动。在一些组织中，项目经理也可能参与与从项目中实现业务利益相关的后续活动。 

# 6、答案：B

解析：控制型 PMO 有权要求项目服从自己的要求，虽然并不直接管理项目。 

# 7、答案：A

解析：不符合 PMBOK 指南中对运营和项目的共同点描述。 

# 8、答案：A

解析：项目生命周期的确定由组织的需求控制决定，与组织的既定方针、项目管理团队生命周期的选择、行业的通用做法有关。 

# 9、答案：A

解析：一类是标准化的工作流程和程序(组织内部的)，一类是过去项目积累下来的共享数据库(知识库)。 

# 10、答案：D

解析：通知适当的管理层存在潜在的利益冲突。一旦他们了解了情况，就可以制定行动计划。PMI的《道德规范》要求项目经理主动向适当的管理层报告利益冲突 

# 11、答案：C

解析：维护和连续运营不是项目的一部分，但需要在项目执行中加以考虑。 

# 12、答案：D

解析：项目执行组织与业务伙伴之间有不属于买卖合同的协议。 

# 13、答案：D

解析：改进现有的业务流程或程序，是流程再造项目。 

# 14、答案：B

14、答案：B

解析：PMO 的主要职能是以各种方式支持项目经理。这可能包括制定和管理项目政策、辅导、指导、培训和监督。这也可能包括对项目之间共享的资源的管理。然而，制定每个项目的详细描述将是处理项目的项目经理的责任，而不是项目管理办公室的责任。 

# 15、答案：A

解析：项目生命周期包括通常顺序排列有时又相互交叉的一系列项目阶段。项目生命周期的阶段划分取决于项目所在组织对项目管控的需要。 

# 16、答案：A

解析：说反了，应该是在项目开始时，运营资源被转移到项目中；而随着项目趋于结束，项目资源被转移到运营中。 

# 17、答案：C

解析：任何项目的商业案例都包括对情况的分析、建议的解决方案和替代解决方案的确定 

# 18、答案、A

解析：在规范的项目管理中，不仅每个项目要经过五大过程组，而且每个阶段可能都要经历五大过程组。 

# 19、答案：C

解析：这就是自由放任的领导风格。 

# 20、答案：A

解析：项目生命周期结构的特征表明：干系人影响力、项目风险与不确定性在项目开始时最大，并在项目的整个生命周期中随时间推移而递减。 

# 21、答案：C

解析：项目组合的范围与组织战略直接相关 

# 22、答案：D

解析：这是一个控制 PMO 的例子。控制性 PMO 执行支持性 PMO 的所有任务，并确保合规性。这不是指令型 PMO，因为它不会直接管理项目。“超级 PMO”是一个虚构的术语。 

# 23、答案：B

解析：项目的特点是循序渐进。它允许项目管理团队随着项目的发展将项目管理到更高的水平。 

# 24、答案：B

解析: 项目发起人通常负责项目商业案例文件的开发和维护。项目经理负责提供建议和监督，以保持项目商业案例、项目管理计划、项目章程和项目效益管理计划的成功措施相互一致，并与组织的目标和目的保持一致。 

# 25、答案：C

解析：标准被视为由公认机构制定并经协商一致确定的准则。相比之下，法规是政府强加的要求。例如，建筑规范就是法规的一个例子 

# 26、答案：C

解析：某个关键干系人不支持项目，不一定就要终止项目。项目经理可以对其进行有效管理 

# 27、答案：B

解析：这是组织过程资产的一部分 

# 28、答案：B

解析：项目之间没有内在联系，只适合当做项目组合进行管理。 

# 29、答案：A

解析：从预测型到迭代和增量型，再到适应型生命周期，对需求明确程度、产品清晰程度的要求逐步降低，项目的风险程度越来越高。 

# 30、答案：B

解析：这是一个约束的例子。制约因素是限制项目管理团队选择的因素。组织任务要求团队的某一部分在不同的地点工作以节省成本，这是项目管理团队需要纳入其规划的一个限制因素。 

# 31、答案：B

解析: 项目发起人通常负责项目商业论证文件的制定和维护。项目经理负责提供建议和见解，使项目商业论证、项目管理计划、项目章程和项目效益管理计划中的成功标准相一致，并与组织的目的和目标保持一致。 

# 32、答案：D

解析：项目管理团队包括项目经理、其他项目管理人员，以及一线的项目执行人员。 

# 33、答案：C

解析：项目成功标准可能包括预期的财务业绩。当一个项目有可能产生财务效益时，可以使用净现值、投资回报率、内部收益率、投资回收期和收益成本比等财务指标，石川分析是一种问题根本原因分析技术，与情况无关。另一方面，EV 和 TCPI 是已经执行的项目的性能衡量指标。 

# 34、答案：B

解析：符合 PMBOK 指南中项目管理的定义。 

# 35、答案：B

解析：项目组合强调服务于组织战略，对项目进行优先级排序，实现组织资源价值最大化。 

# 36、答案：B

解析：规划的主要作用就是明确目标和路线，识别最佳工作方法。 

# 37、答案：C

解析：项目集中的项目必然有内在联系。 

# 38、答案：C

解析：六个相互竞争的项目制约因素是范围、质量、进度、成本、资源和风险。采购不是项目竞争的制约因素。 

# 39、答案：B

解析：问题和缺陷管理数据库被认为是组织过程资产的一部分。这些数据库通常包含历史问题和缺陷状态、控制信息、问题和缺陷解决方案以及行动项目结果。 

# 40、答案：B

解析：组织的商业价值中包括全部的有形价值和无形价值。 

# 41、答案：A

解析: 项目相关方是积极或消极参与项目或其利益可能受项目实施完成的积极或消极影响的个人或组织。 

# 42、答案:。D

解析：项目中应尽早进行变更，以避免额外的成本和延误。干系人的影响力在项目开始时很高，而在项目结束时很低。相比之下，在项目开始时对项目进行更改的成本较低，而在项目结束时成本较高。 

# 43、答案：B

解析：项目经理是为项目执行组织实现项目目标的那个人。 

# 44、答案：B

# 45、答案：A

# 46、答案：C

解析：无论是支持型、控制型还是指令型的PMO，都需要为项目提供咨询和指导，这是PMO最基本的职责。 

# 47、答案：A

解析：典型的业务分析师的职责包括确定问题，确定业务需求，并为满足这些需求推荐可行的解决方案 

# 48、答案：C 

解析：项目组合管理关注：项目之间的优先顺序以及由此而形成的资源分配的优先顺序。 

# 49、答案：C

# 50、答案：C

# 51、答案：B

# 52、答案：B

# 53、答案：C

# 54、答案：C
